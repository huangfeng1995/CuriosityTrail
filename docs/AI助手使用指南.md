# AI 助手使用指南

## 概述

Curiosity Trail 现在集成了强大的 AI 助手功能，支持多种 AI 服务提供商，包括本地模型！

## 快速开始

### 1. 使用本地模型（推荐，免费）

如果你已经下载了 Qwen2 或其他 GGUF 格式的模型，这是最简单的方式：

1. 打开应用，进入「系统设置」
2. 点击「AI 助手配置」标签页
3. 在「服务选择 & 状态」中，确认「当前聊天服务」设置为 `本地模型 (LlamaCpp)`
4. 在「本地模型 (LlamaCpp)」标签页中，确认模型路径正确（默认已设置为 `/Users/openclaw/Downloads/qwen2_05b_int4.gguf`）
5. 点击「保存配置」
6. 前往「AI 研究助手」开始对话！

### 2. 使用 Ollama

如果你已经安装并运行了 Ollama：

1. 进入「AI 助手配置」→「Ollama」标签页
2. 确认服务地址为 `http://localhost:11434`
3. 选择你想用的模型
4. 在「服务选择」中设置聊天服务为 `Ollama`
5. 保存配置

### 3. 使用 MiniMax API（需要 API Key）

如果你想使用云端的高质量模型：

1. 在 MiniMax 开放平台申请 API Key 和 Group ID
2. 进入「AI 助手配置」→「MiniMax API」标签页
3. 填入你的 API Key 和 Group ID
4. 在「服务选择」中设置聊天服务为 `MiniMax API`
5. 保存配置

## 服务对比

| 服务 | 优点 | 缺点 | 推荐场景 |
|------|------|------|---------|
| **本地模型 (LlamaCpp)** | 完全免费，无需联网，数据隐私 | 需要下载模型，性能取决于硬件 | 日常使用，重视隐私 |
| **Ollama** | 简单易用，模型选择多 | 需要安装运行 Ollama | 已有 Ollama 的用户 |
| **MiniMax API** | 高质量翻译和对话，响应快 | 需要付费，联网使用 | 需要高质量结果的场景 |

## 翻译功能

翻译服务与聊天服务可以独立配置：

1. 在「AI 助手配置」→「翻译设置」标签页中设置默认语言
2. 在「服务选择」标签页中，可以选择翻译服务（默认与聊天服务相同）

### 翻译场景

- **报告编辑器**：点击「翻译全文」按钮
- **文献库**：在每个文献卡片上点击「翻译」
- **AI 助手**：在 AI 回复下方点击「翻译」

## 配置说明

### 本地模型 (LlamaCpp) 设置

- **模型文件路径**：你的 GGUF 模型文件的绝对路径
- **温度参数**：0.1-0.3 适合翻译，0.7-1.0 适合创意写作
- **上下文大小**：根据模型支持设置，一般 2048-8192
- **GPU 层数量**：-1 表示使用所有可用层（推荐），0 表示仅使用 CPU

### 高级设置

- **最大生成长度**：控制单次回复的长度
- **系统提示词**：自定义 AI 的角色设定

## 故障排除

### 本地模型无法加载

1. 确认模型文件路径正确（绝对路径）
2. 检查文件权限
3. 尝试降低上下文大小

### 翻译质量不好

1. 尝试使用更大的模型
2. 调整温度参数（降低至 0.1-0.3）
3. 考虑使用 MiniMax API

### 页面加载缓慢

1. 本地模型首次加载会较慢，之后会变快
2. 使用更小的模型（如 0.5B 或 1.5B）
3. 确保有足够的内存

## 推荐模型

对于翻译和研究助手任务：

- **Qwen2 0.5B**: 最小最快，适合简单任务
- **Qwen2 1.5B/7B**: 平衡速度和质量
- **Llama 3.2 1B/3B**: 通用能力强
- **Mistral 7B**: 高质量，需要更好硬件
