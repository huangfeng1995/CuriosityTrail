# LlamaCpp (GGUF) 模型使用指南

## 概述

本项目现在支持直接使用 GGUF 格式的本地模型进行翻译，无需依赖 Ollama。

## 快速开始

### 1. 安装依赖

```bash
cd web/server
npm install
```

### 2. 准备模型

确保你的 GGUF 模型文件位于：
`/Users/openclaw/Downloads/qwen2_05b_int4.gguf`

或者，你可以在翻译设置页面修改模型路径。

### 3. 配置翻译服务

1. 启动应用
2. 进入「系统设置」→「翻译设置」
3. 选择「LlamaCpp (GGUF)」标签页
4. 确认模型路径正确
5. 在「使用的翻译服务」中选择「LlamaCpp (GGUF)」
6. 保存设置

### 4. 开始翻译

现在你可以使用：
- 报告编辑器的「翻译全文」功能
- 文献库的文档翻译功能
- AI 助手回复的翻译功能

## 支持的模型

任何 GGUF 格式的模型都应该可以工作，包括：
- Qwen 系列 (推荐)
- Llama 系列
- Mistral 系列
- 其他...

## 使用提示

1. **模型大小**：对于翻译任务，小模型 (0.5B-7B) 通常已经足够
2. **性能**：首次加载会较慢，后续会更快
3. **GPU 加速**：`node-llama-cpp` 会自动检测并使用可用的 GPU
4. **温度参数**：翻译时建议使用较低的温度 (0.1-0.5)

## 故障排除

### 模型文件找不到
- 确保模型路径是绝对路径
- 检查文件权限

### 翻译质量不佳
- 尝试使用更大的模型
- 调整温度参数
- 考虑使用其他翻译服务

### 性能问题
- 使用量化模型 (int4, int8)
- 减少 `max_tokens` 限制
- 确保有足够的内存

## 对比其他翻译服务

| 服务 | 特点 | 推荐场景 |
|-----|------|---------|
| MiniMax API | 高翻译质量，付费 | 生产环境，高质量需求 |
| Ollama | 需要本地模型服务 | 已安装 Ollama 的用户 |
| LlamaCpp (GGUF) | 直接使用模型文件，灵活 | 有 GGUF 模型的用户 |
